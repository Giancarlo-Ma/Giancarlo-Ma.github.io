Thread creation

The following is the thread creation interface for POSIX.

Arguments

  • thread: a pointer to a structure of type pthread_t, use the structure to interact with this thread.
  • attr: specify attributes the thread may have. For example, set the stack size or information about scheduling priority of the thread. A call to pthread_attr_init is needed to initialize an attribute. Most cases, default is fine.
  • start_routine: A function which the thread start running in. It’s a function pointer in C. For example, see the following to understand how to use it.
  • arg: the argument to be passed to the function where the thread begins execution. Having a void pointer as argument allows us pass any type of parameter to the start_routine, have it as return value allows us return any type of result.

Thread Completion

When we want to wait for a thread to complete, we call the routine pthread_join.

Arguments

  • thread: specify which thread to wait for. It’s initialized by the thread creation routine(pass a pointer to it as an argument to pthread_create).
  • value_ptr: a pointer to the return value u expect to get back. The routine can return anything, so a void pointer. Routine changes the value of the argument, we need to pass a pointer to that value, not just value itself.

Example

  • often times we don’t have to do all of the packing and unpacking of arguments. Just pass NULL to the routine.
  • If we pass a single value(an int), we don’t have to package it up as an argument.
